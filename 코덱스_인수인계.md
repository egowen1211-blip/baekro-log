## 3. 현재 구현 완료 기능

### 3.1 기본 조사 흐름

1. 조사 시작
2. 참여자 입력
3. 조사 구간 선택
4. GPS 측정
5. 조사 중 종 기록 반복
6. 조사 종료
7. 조사 소감 및 특이사항 작성
8. 최종 제출
9. 학생 기록 탭에서 제출 기록 확인

### 3.2 자동 기록 항목

- 조사 일자
- 조사 시작 시각
- 조사 종료 시각
- 날씨
- 시작 위치
- 종료 위치
- 조사 이동 경로
- 이동 거리
- 관찰 시각
- 관찰 GPS
- GPS 정확도

### 3.3 학생 입력 항목

- 참여자
- 조사 구간
- 종명
- 개체수
- 출현지역·서식 특성
- 기타 직접 입력
- 사진
- 조사 소감 및 특이사항

### 3.4 사진

- 스마트폰 촬영 우선
- `capture="environment"` 유지
- 긴 변 1600px 이하로 압축
- JPEG 품질 약 0.8
- IndexedDB에 저장
- 관찰 기록과 사진 ID 연결
- 저장 사진이 없거나 깨졌을 때 깨진 이미지 아이콘 미표시

### 3.5 관찰 카드

관찰 카드에 표시되는 내용:

- 관찰 사진
- 종명
- 녹색 개체수
- 관찰 시각
- GPS 좌표
- GPS 정확도
- 위치 입력 방식
- 출현지역·서식 특성
- 관찰 위치 지도 이미지
- 진행 중일 때만 삭제 버튼

`큰 지도에서 위치 보기`는 제거됨.

### 3.6 관찰 위치 지도 이미지

- 관찰 카드에는 인터랙티브 지도 대신 지도 캡처형 이미지 사용
- OpenStreetMap 타일을 canvas에 조합해 이미지 생성
- 빨간 관찰 위치 표시
- GPS 정확도 반경 표시
- OpenStreetMap 출처 표시
- 생성 이미지는 IndexedDB에 저장
- 기존 관찰 기록에 지도 이미지가 없으면 열 때 자동 생성
- 지도 타일 캡처 실패 시 좌표·마커 기반 대체 이미지 생성

### 3.7 조사 이동 경로 지도

- Leaflet + OpenStreetMap
- 조사 중 이동 경로 표시
- 시작 위치 표시
- 관찰 위치 마커 표시
- 제출 완료 기록에서는 종료 위치 표시
- 메인 지도는 계속 인터랙티브 지도 사용

### 3.8 GPS 정확도 처리

기본 기준:

- 목표 정확도: ±5m
- 허용 기준: ±10m

GPS가 ±10m 이내면 자동 사용.

GPS가 ±10m를 초과하면 다음 선택지 제공:

1. GPS 다시 측정
2. 지도에서 위치 지정
3. 정확도 초과 GPS로 저장

정확도 초과 저장 시 사유 선택:

- 건물 또는 나무로 인한 신호 불량
- 이동 중 기록
- 지도 사용 불가
- 기타

저장 항목:

- 자동 측정 원본 위도
- 자동 측정 원본 경도
- 자동 측정 원본 정확도
- 최종 위도
- 최종 경도
- 위치 입력 방식
- 위치 입력 메모

위치 입력 방식:

- `gps`
- `map`
- `low_accuracy`

### 3.9 지도 위치 지정

별도의 두 번째 Leaflet 지도는 제거함.

현재 방식:

- 기존 `조사 이동 경로` 지도를 재사용
- 지도 중앙에 빨간 십자 표시
- 지도를 움직여 관찰 대상 위치 지정
- 하단 고정 도구막대 사용
- 버튼:
  - 닫기
  - 측정 GPS
  - 이 위치로 지정

이 방식은 아이폰 Safari에서 별도 지도 흰 화면 문제가 발생해 변경한 것임.

### 3.10 조사 종료 및 최종 제출

조사 종료 시 다음 요약 표시:

- 조사 구간
- 조사 시간
- 관찰 종 수
- 총 개체수
- 관찰 기록 수
- 사진 수
- 이동 거리

소감:

- 항목명: `조사 소감 및 특이사항`
- 최대 500자
- 글자 수 표시
- 작성 중 자동 임시 저장

버튼:

- 조사 계속하기
- 최종 제출

최종 제출 후:

- 학생이 관찰 기록 추가 불가
- 관찰 기록 삭제 불가
- 조사 기록 삭제 불가
- 제출 상태 표시
- 제출 시각 표시
- 소감 표시

상태값:

- 진행 중: `진행중`
- 제출 완료: `submitted`
- 보조 필드: `submissionStatus: submitted`

### 3.11 제출 완료 화면

제출 완료 기록을 열면:

- 현재 GPS 카드 숨김
- 현재 위치 버튼 숨김
- 전체 경로 버튼 숨김
- 조사 종료 정보 카드 표시
- 총 이동 거리 표시
- 조사 종료 위치 표시
- 경로 지도에 종료 위치 마커 표시

### 3.12 학생 기록 탭

학생 기록 탭에는 다음만 표시:

- 최종 제출 기록 목록
- 제출 상태
- 조사 결과 요약
- 소감 요약
- 제출 내용 보기

제거된 기능:

- CSV 내보내기
- JSON 백업

이 기능들은 교사 화면에만 구현할 예정.

## 4. 현재 저장 구조

### 4.1 LocalStorage

현재 조사 초안 및 제출 기록 메타데이터를 저장.

주요 개념:

- draft
- history
- settings
- 조사 구간 목록
- 교사 PIN 설정

정확한 키 이름은 `index.html` 내부 `STORAGE` 상수 확인.

### 4.2 IndexedDB

다음 바이너리 데이터 저장:

- 관찰 사진
- 지도 캡처 이미지

현재는 브라우저/기기 로컬에만 존재.

주의:

- 브라우저 사이트 데이터 삭제 시 사진과 지도 이미지 삭제 가능
- 다른 기기에서는 볼 수 없음
- JSON 메타데이터만으로 사진 blob 복원 불가

### 7.4 동기화 정책

현장 인터넷 불안정 고려.

권장 흐름:

1. 학생이 종 기록 저장
2. LocalStorage/IndexedDB 즉시 저장
3. 온라인이면 Supabase에 `draft` 자동 동기화
4. 오프라인이면 대기 큐 저장
5. 온라인 복귀 시 재전송
6. 최종 제출 시 모든 데이터 업로드 완료 확인
7. `status = submitted`
8. 교사 화면에 노출

### 7.5 RLS 방향

학생:

- 자신의 profile 조회/수정
- 자신의 survey_sessions 조회/생성
- 자신의 draft 수정
- submitted 이후 수정 제한
- 자신의 observation 및 media만 접근

교사:

- 소속 project의 submitted/revision_requested/approved 조회
- review_status 및 teacher_comment 수정
- 수정 요청/확인 완료 처리
- 설정 데이터 관리

주의:

- 프론트엔드에는 publishable key만 사용
- service_role key 절대 포함 금지
- 모든 public 테이블 RLS 활성화

## 9. 반드시 유지할 기술/화면 정책

- 학생 화면은 모바일 세로 우선
- 관찰 카드에 인터랙티브 지도 금지

- 학생 화면에는 CSV/JSON 내보내기 없음
- 교사 실시간 위치 추적 없음
- 최종 제출 전 소감 필수
- 최종 제출 후 학생 수정 잠금
- GPS ±10m 초과 시 지도 지정 또는 사유 저장
- 원본 GPS와 최종 좌표 모두 보존
- 사진/지도 이미지는 추후 Supabase Storage로 이전

## 10. 현재 알려진 주의사항

1. OpenStreetMap 타일 canvas 캡처는 브라우저 CORS 환경에 따라 실패할 수 있음.
2. 실패 시 대체 지도 이미지가 생성되도록 구현됨.
3. iPhone Safari에서 별도 Leaflet 인스턴스가 흰 화면으로 표시된 전력이 있음.
4. 위치 지정은 메인 이동 경로 지도를 재사용하도록 변경됨.
5. `capture="environment"`를 써도 iPhone Safari는 사진 보관함 선택을 보여줄 수 있음.
6. 현재 사진·지도 이미지는 IndexedDB 로컬 저장임.
7. 제출 완료 데이터도 아직 Supabase에 올라가지 않음.
8. 교사 설정 PIN은 임시 구조임.
